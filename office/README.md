# Office

This folder is the active control layer for Aurora Sentinel Scanner.

It is intentionally small.

## Purpose

The office layer exists to control implementation without turning the repository into paperwork.

It owns:
- control law
- task status
- locked decisions
- work logging
- SHA logging

It does not own:
- MT5 product code
- runtime design law
- archive storage
- handoff clutter
- wave packet sprawl

## Active office files

- `OFFICE_CANON.md`
- `TASK_BOARD.md`
- `DECISIONS.md`
- `WORK_LOG.md`
- `SHA_LEDGER.md`

## Archive interpretation

The old archive office proved that the repo benefits from:
- a task board
- a decision log
- a master log
- a SHA ledger
- stage-gated control

The old archive office also proved that too many HQ, handoff, review, and wave files create drag.

This office keeps the useful control surfaces and drops the clutter.
