# ASC Include Root

This is the future include root for ASC product code.

## Intended domains
- `Common`
- `Discovery`
- `Runtime`
- `Layers`
- `Publication`
- `Presentation`

Each domain owns a meaning.
No domain should become a dumping ground for unrelated logic.


## Menu alignment law
- future menu/input ownership must stay aligned with these domains
- runtime-owned settings belong with Runtime
- publication-owned settings belong with Publication
- later layer settings should not be hidden inside unrelated groups
