# QUANT / PYTHON / MACHINE FOUNDATION

## PURPOSE

This file gives Aurora a dedicated quant / Python / machine foundation lane.

It exists so the uploaded quantitative books do not remain scattered influences on the strategy layer.
Instead, they become an explicit branch that can later support:
- rule formalization
- backtesting discipline
- statistical testing
- optimization
- machine-learning-assisted market classification
- future Python implementation bridges

This lane is not meant to replace chart-first doctrine.
It is meant to discipline and extend it.

---

# 1. ROOT LAW

Aurora should not confuse:
- chart reading
with
- quant validation

The chart-first side can generate hypotheses.
The quant/Python side should test, translate, compare, rank, and reject them.

This is the main function of the quant/machine lane.

# 4. FIRST QUANT QUESTIONS AURORA SHOULD ASK

For any serious family or pattern later, the machine lane should help answer:
- can this be expressed as rules?
- what variables define it?
- what evidence type is available?
- how should it be backtested?
- what is the relevant risk/return framing?
- what are the main overfitting dangers?
- what extra data surfaces improve it materially?
