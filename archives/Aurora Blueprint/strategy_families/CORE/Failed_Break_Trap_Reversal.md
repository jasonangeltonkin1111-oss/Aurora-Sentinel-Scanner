# FAILED BREAK / TRAP REVERSAL

## PURPOSE

This file defines the core Aurora Failed Break / Trap Reversal family.

It is written as a family-definition file, not as a final trading ruleset.

Its job is to make the family:
- structurally clear
- habitat-clear
- anti-confusion clear
- multi-surface aware for later wrapper / EA use

---

# 1. FAMILY NAME

## Canonical name
Failed Break / Trap Reversal

## Closely related labels
- trap family
- failed-state family
- failed break reversal
- reclaim reversal

---

# 2. CANONICAL IDEA

Participate when an attempted directional move loses acceptance, reclaims prior structure, and reversal / return-to-structure logic becomes dominant.

This family is not about fading any breakout.
It is about structural failure that becomes clear enough that reversal is now stronger than continuation.

---

# 3. NATIVE HABITAT

Primary native habitats:
- Failed State / Trap
- Failed Break / Trap State
- false commitment reverting back through initiating structure

This family belongs where:
- a visible break attempt occurred
- continuation could not hold
- rejection/reclaim back through the initiating zone becomes structurally dominant

---

# 4. ANTI-HABITAT

This family should not be treated as native in:
- unresolved breakout conditions
- fresh accepted release that still holds
- unstable trap-on-trap chaos with no clear reclaim hierarchy
- pure balance rotation with no failed directional attempt

---

# 5. STRUCTURAL SIGNATURES

Typical structural signatures include:
- visible break attempt
- inability to maintain acceptance beyond the break
- reclaim of prior structure
- responsive logic stronger than continuation logic
- weakening trust in the original directional attempt

---

# 6. MAIN COMPETING INTERPRETATIONS

Failed Break / Trap Reversal must be distinguished from:
- Breakout / Compression Release
- Trend Continuation
- Balance Rotation / Range Mean-Reversion
- unresolved transition where the break has not yet clearly failed

The family is only valid when failure is clearer than continuation.

---

# 7. MAIN ANTI-CONFUSIONS

Do not activate this family when:
- the break is still unresolved
- rejection is not yet structurally clear
- multiple alternating traps make the environment unstable
- the move is only ordinary pullback rather than structural failure

---

## Required native data surface

- live trust / spread / liquidity degradation cues
- session/open conditions that make fake breaks more common or more meaningful
