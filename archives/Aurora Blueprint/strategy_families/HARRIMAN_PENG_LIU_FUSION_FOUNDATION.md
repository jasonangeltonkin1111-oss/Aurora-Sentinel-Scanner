# 4. GOLDEN SEPARATION RULE

Do not merge visual pattern recognition, machine learning, indicators, optimization, and strategy testing into one undifferentiated strategy soup.

Instead:

## Harriman layer should mostly
- identify
- classify
- narrate
- interpret
- locate key zones
- generate hypotheses

## Peng Liu layer should mostly
- formalize
- parameterize
- test
- compare
- rank
- reject weak edges

That means Aurora should reason like this:
1. I see a structure
2. classify the state/surface
3. identify the family
4. attach the likely pattern child
5. translate the setup into measurable rule candidates
6. define how it should be tested

# 5. FIVE-LAYER WRAPPER LOGIC

The trader-chat guidance supports a five-layer mental architecture.
Aurora should preserve it.

## Layer 1 — Market Reading Layer
Main source:
- Harriman

Purpose:
- read the chart
- determine context
- identify regime

Outputs:
- trend / range / volatility / compression / reversal risk
- structure
- key zones
- candle behavior / price behavior

## Layer 2 — Strategy Family Layer
Main source:
- Harriman + Peng Liu

Purpose:
- map the setup into a strategy family rather than jumping straight into trade logic

## Layer 3 — Strategy Card Layer
Main source:
- both books fused

Purpose:
- give Aurora reusable strategy cards instead of improvised one-off opinions

## Layer 4 — Quant Check Layer
Main source:
- Peng Liu

Purpose:
- translate the setup to rules
- define variables
- define testing logic
- define evidence quality and risk controls

## Layer 5 — Output / Routing Layer
Main source:
- Aurora wrapper discipline

Purpose:
- produce chart interpretation, family match, card match, rule candidate, risk note, confidence, and invalidation clearly
