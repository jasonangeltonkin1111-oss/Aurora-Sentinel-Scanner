# 2. WHY THIS FOLDER EXISTS

The cards folder exists so strategy families can evolve in a controlled way.

Each card should be:
- stable enough for wrapper/EA planning
- honest enough to preserve provisionality
- enrichable as more books are processed
- organized enough that no strategy-family knowledge gets lost

---

# 3. EVOLUTION RULE

When future books add new depth, cards should usually be updated by:
- sharpening habitat definitions
- improving invalidation logic
- adding supporting data surfaces
- improving family rejection conditions
- adding pattern children or testing notes

Cards should rarely be rewritten from scratch unless the family itself is redefined.

This protects continuity while still allowing strategy knowledge to deepen over time.

---

# 4. CURRENT JUDGMENT

Aurora now has a dedicated strategy-card layer separated cleanly from:
- registry files
- family-definition files
- broader doctrine files

This makes the strategy side more organized, more future-buildable, and less likely to lose hard-won family knowledge as the library grows.
