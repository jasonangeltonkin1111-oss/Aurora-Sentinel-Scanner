# AURORA MASTER TASK MAP 001

## PURPOSE

This file is the first explicit full-project task map for Aurora Blueprint.

It exists to answer, in one place:
- what the main task actually is
- what is already sufficiently built
- what still needs to be done
- what is a main-track task versus a side quest
- what order should be followed from here

This file is intentionally blunt.

Aurora’s main task is still:
- extract the book library into a complete big-picture doctrine universe

Everything else is secondary unless it clearly supports that main task.

That means:
- extraction is the main track
- continuity, consolidation, and deepening are support tracks
- support tracks should not outrun big-picture extraction

---

# 1. ROOT LAW

The project should now be operated under this priority law:

## Main track
Build the full big picture from the books.

## Support tracks
Only do control, consolidation, deepening, wrapper-prep, and EA-safe translation when they are needed to:
- preserve extraction truth
- prevent drift
- make the next extraction wave safer
- compress already-extracted truth after enough of the picture exists

If support work begins consuming the majority of project energy before the larger source universe is covered, support work has become a side-quest trap.

# 3. WHAT THE MAIN TASK STILL REQUIRES

To actually finish the big picture, Aurora still needs broader source coverage.

That means:
- continue extracting the remaining library
- preserve each new source set strongly enough in one bounded campaign
- keep the ledger honest
- avoid repeated re-upload dependence

The big picture is not complete until the remaining meaningful library clusters have been extracted, classified, and placed.

---

# 4. MAIN-TRACK TASK MAP

These are the main tasks from here.

## MAIN TASK A — Finish remaining meaningful book extraction

Goal:
- complete broad source coverage of the remaining meaningful library clusters

Required sub-tasks:
1. identify remaining unprocessed or only intake-tracked books
2. batch them into sensible source sets
3. run bounded source-pass extraction
4. create doctrine destinations only where needed
5. update ledger truth
6. avoid re-upload repetition

Success condition:
- the remaining meaningful source universe is no longer mostly “not yet translated” or “reference-intake only”

---

## MAIN TASK B — Preserve the big-picture map of the library

Goal:
- know what the total source universe is, what role each book plays, and what remains

Required sub-tasks:
1. keep the master book index honest
2. keep the ledger honest
3. classify each remaining book as:
   - TRANSLATE
   - REFERENCE ONLY
   - DO NOT USE
4. mark whether the book’s current-wave value is actually captured yet

Success condition:
- the project can answer clearly what books still matter and why

---

## MAIN TASK C — Only after enough extraction, resume heavier synthesis

Goal:
- return to consolidation/deepening only after broader extraction gives the system its fuller big picture

Required sub-tasks later:
1. merge newly extracted source truth into doctrine surfaces
2. add wrapper/ranking/EA-safe translation layers
3. deepen only where the expanded source universe justifies it

Success condition:
- synthesis happens from broad source truth, not from a partial library slice

# 6. SIDE-QUEST MAP

These are useful, but they are not the main task unless they directly support extraction.

## Side Quest 1 — More control-layer generations
Only do this if repo truth materially changes and recovery would otherwise break.

## Side Quest 2 — More consolidation passes
Only do this when:
- a lot more source truth has been extracted, or
- a specific doctrine surface is blocking extraction or usability

## Side Quest 3 — More deepening passes
Only do this when:
- the deeper pass unlocks cleaner future extraction or classification, or
- a specific doctrine ambiguity is blocking correct downstream work

## Side Quest 4 — Wrapper / ranking / EA-safe translation layers
Important later, but not the main track until the big picture is broader.

## Side Quest 5 — Pretty restructuring or file polishing
Low priority unless it prevents confusion or data loss.
