# 2. QUANT / MACHINE STRENGTHENING ADDITIONS

## 2.1 Upstream-data-structure rule

### Strengthening statement
The machine lane should begin with event-aware data structures and sampling discipline rather than jumping straight to models.

### Why this is justified
AFML organizes the workflow from financial data structures and sampling features forward into labeling, weighting, validation, and allocation.

### Canonical implication
Aurora’s machine lane should preserve upstream discipline as part of model quality, not as external preprocessing trivia.

---

## 2.2 Meta-labeling rule

### Strengthening statement
Direction and participation / size should be allowed to live as separate modeling objects rather than being collapsed into one output by default.

### Why this is justified
AFML explicitly separates side from size and develops meta-labeling as a way to improve selectivity and sizing discipline.

### Canonical implication
Aurora’s machine lane should preserve the option to decouple entry-side logic from confidence / sizing logic.

---

## 2.3 Uniqueness-aware-training rule

### Strengthening statement
Training weights should reflect overlap and uniqueness rather than assuming every labeled event contributes equally independent evidence.

### Why this is justified
AFML explicitly treats concurrent labels, average uniqueness, return attribution, and time decay as core issues in financial ML.

### Canonical implication
Aurora’s machine lane should preserve non-uniform weighting when examples share the same realized path.

---

## 2.4 State-model-branch rule

### Strengthening statement
Hidden-state / regime-switching models are a legitimate branch inside the machine lane when evidence supports distinct persistent behaviors across time.

### Why this is justified
The Markov-switching note formalizes hidden-state processes with transition probabilities, persistence, mean switching, and variance switching as a practical nonlinear alternative to one stationary template.

### Canonical implication
Aurora’s machine lane should preserve state/regime modeling as a bounded branch rather than treating it as a curiosity.

---

## 2.5 Tested-switching rule

### Strengthening statement
State-model adoption should be gated by estimation and test pressure rather than by intuitive regime storytelling.

### Why this is justified
The Markov-switching note includes explicit estimation procedures and hypothesis tests for switching structure.

### Canonical implication
Aurora’s machine lane should preserve switching models as evidence-gated tools, not narrative shortcuts.

AFML explicitly places bet sizing, strategy risk, and machine-learning asset allocation inside the same financial-ML architecture.

## 2.7 Structural-break-and-microstructure-awareness rule

### Strengthening statement
The machine lane should remain aware of structural breaks and market-plumbing information instead of pretending the feature matrix is stationary and context-free.
