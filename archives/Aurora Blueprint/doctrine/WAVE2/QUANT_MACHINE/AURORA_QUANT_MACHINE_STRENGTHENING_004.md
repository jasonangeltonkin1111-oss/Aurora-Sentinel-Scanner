# 2. QUANT / MACHINE STRENGTHENING ADDITIONS

## 2.1 Allocation-equals-relative-leverage rule

### Strengthening statement
Allocation and leverage should be treated as tightly connected dimensions rather than as fully separate decisions.

### Why this is justified
Vince explicitly states that allocation is the relative leverage between portfolio components and therefore that allocation and leverage refer to the same exercise in this framework. fileciteturn98file0

### Canonical implication
Aurora’s quant/machine lane should preserve tighter conceptual unity between allocation and leverage when later portfolio/ranking logic is built.

## 2.2 Drawdown-aware-portfolio rule

### Strengthening statement
Portfolio construction should be evaluated under drawdown-aware constraints rather than only under unconstrained optimality criteria.

### Why this is justified
The preface and introduction repeatedly emphasize drawdown and lower absorbing barriers on equity as the real-world constraint that formal portfolio logic must answer. fileciteturn98file0

### Canonical implication
Aurora’s machine lane should preserve drawdown-aware portfolio framing whenever later capital-allocation logic is built.

## 2.3 Geometric-growth-allocation rule

### Strengthening statement
Allocation frameworks should remain aware of geometric-growth and reinvestment effects rather than treating repeated bets as static arithmetic expectations.

### Why this is justified
The contents place reinvestment of returns, geometric growth, and optimal-f before later portfolio-construction chapters. fileciteturn98file0

### Canonical implication
Aurora’s machine lane should preserve geometric-growth awareness when comparing portfolio constructions and allocations.

---

## 2.4 Anti-variance-only-optimization rule

### Strengthening statement
Variance-only portfolio optimization is incomplete when leverage, path dependence, and drawdown constraints are load-bearing.

### Why this is justified
The book explicitly progresses beyond classical portfolio construction and mean-variance geometry into the leverage-space model while also critiquing variance as a practical proxy for risk. fileciteturn98file0

### Canonical implication
Aurora’s machine lane should preserve bounded skepticism toward variance-only optimization whenever richer allocation logic is justified.
