This file is the entry index for the Aurora patterns folder.

The patterns folder is the organized child layer below:
- doctrine
- strategy families
- strategy family cards

# 2. WHY THIS FOLDER EXISTS

The patterns folder exists so pattern knowledge can evolve in a controlled way.

Pattern cards should remain:
- attached to parent families
- honest about state/surface dependencies
- enrichable as more books are processed
- clear enough for future machine translation

This prevents the pattern layer from becoming a giant disconnected pattern list.

---

# 3. EVOLUTION RULE

When future books add more depth, the pattern layer should usually be enriched by:
- sharpening existing pattern cards
- adding narrower child cards under existing groups
- improving invalidation/failure logic
- improving misread / competitor distinctions
- adding richer data-surface dependencies

Do not add new patterns without attaching them to:
- a parent family
- a habitat
- a meaning
- anti-confusions or competing interpretations
