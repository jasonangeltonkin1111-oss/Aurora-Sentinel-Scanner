03 — HYBRID 5D GROWTH LAW (AURORA + ISS GOVERNANCE)
================================================================================
A change is allowed only if:
- it improves at least TWO dimensions among (1)-(4), AND
- it does NOT materially degrade Functional Viability (5).

(1) QUALITY: clearer thesis, cleaner invalidation, better regime fit, fewer false positives.
(2) STABILITY: fewer contradictions/oscillation, better determinism, bounded complexity (DCI), cleaner auditability.
(3) REALISM: better feasibility (session/event/spread), better data-truth constraints, fewer “fiction outputs.”
(4) COVERAGE / GROWTH: broader supported inputs / assets / vault hooks WITHOUT loosening hard floors.
(5) FUNCTIONAL VIABILITY (HARD GUARD): must remain usable for daily market study.

UPWARD COMPLEXITY RULE
- If a change grows “up” (more logic), it must also grow “sideways” (coverage hooks) AND add stability, AND pass Functional Viability.

NO “GROWTH BY LOOSENING”
- Frequency gains must come from breadth + TF scaling + routing, not breaking safety floors.

VIABILITY CHECKPOINT (HARD; DEV STAGE)
- All major expansions must state qualitative effects on scenario density, DCI, and operator friction.
- Dev overseers (Section 23) enforce this checkpoint in dev stage.

12.1 — CONFIDENCE PERCENTAGE MODEL + ISS/MACRO MODIFIER (HARD)
================================================================================
PURPOSE
- Replace confidence_band with confidence_percentage (x.x%) as the single surface confidence metric.
- Confidence percentage is a STRUCTURAL COHERENCE proxy (not win probability).
- ISS/macro may adjust confidence within strict bounds; structure remains alpha owner.

DEFINITIONS
- SCI_structural: internal structural coherence score (0..1), derived from CSI components (structure-dominant).
- ISS_modifier: bounded additive modifier applied AFTER SCI_structural is computed (cannot alter direction).

STRUCTURAL INPUTS (dominance order; no tuned numbers in blueprint)
- StructureIntegrity (dominant)
- TacticalAsymmetryIntegrity (high)
- LiquidityIntegrity (medium)
- VolatilityIntegrity (medium)
- RegimeAlignment (medium)
- ProjectionEfficiency (medium)
- ContextIntegrity (ISS/macro) (low; bounded; modifier only)

MODEL (HARD)
1) Compute SCI_structural from structural inputs only (ISS excluded).
2) Map SCI_structural to confidence_percentage via bounded ranges:
   - Forced tie-break baseline: 48.0% – 62.0%
   - Organic with HTF friction: 58.0% – 72.0%
   - Strong alignment: 66.0% – 82.0%
   - Extreme chaotic/dislocated: 45.0% – 58.0%
   Hard ceiling: 84.5%
   Hard floor: 44.0%
   Rounding: 1 decimal place (nearest tenth).
3) Apply ISS/macro modifier ONLY if ISS_CONFIDENCE_MODIFIER_POLICY=ON and ISS/macro context is present:
   - ISS_modifier must be bounded:
     - Positive (alignment): +0.0% to +4.0%
     - Negative (conflict/risk/friction): -2.0% to -8.0%
   - Negative impact is allowed to be stronger than positive impact.
4) Apply hard caps AFTER ISS modifier:
   - If forced_tiebreak_flag == TRUE: cap at 62.0%
   - If extreme_state in {EXTREME_CHAOTIC, EXTREME_DISLOCATED}: cap at 55.0%
   - If extreme_state == EXTREME_STRUCTURED: cap at 68.0%
   - If HTF friction proximity flag present (from structure/perception): apply penalty (bounded) and do not exceed 72.0% unless HTF friction clears.
5) Set confidence_modifier_source:
   - STRUCTURAL_ONLY if no ISS modifier applied
   - ISS_ADJUSTED if ISS modifier applied
   - UNDEFINED if cannot be determined (No-Invention Clause)

DISPLAY LAW (HARD)
- SIAM surface displays confidence_percentage only.
- confidence_modifier_source is written to CSV always; MAY be omitted from SIAM surface.
- confidence_band is not rendered on SIAM surface; legacy policies may exist but do not appear as outputs.
