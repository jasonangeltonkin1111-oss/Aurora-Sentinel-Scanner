# ARCHIVES INDEX

This file is the permanent navigation map for the full `archives/` source layer.

The archive is immutable source memory.
It preserves historical blueprints, legacy systems, extracted code references, maps, and the Aurora research library.

If any active system document conflicts with archive material, the archive is still preserved unchanged; active meaning is resolved in `blueprint/`, not by editing archive content.

---

# 1. HARDLOCK STATUS

`archives/` is permanently preserved.

Allowed:
- additive markdown navigation files
- read-only analysis
- blueprint translation outside the archive

Forbidden:
- deleting archive files or folders
- renaming archive files or folders
- changing archive source content
- restructuring archive source directories

See:
- `archives/ARCHIVE_IMMUTABILITY.md`

# 2. TOP-LEVEL ARCHIVE MAP

## `archives/Aurora/`
Aurora research library and core long-horizon conceptual archive.
Contains books, PDFs, EPUBs, MOBIs, and key text blueprints for future knowledge extraction.

Use this for:
- future-phase research extraction
- concept mining
- rulecard source identification
- long-range design context

Do not use it to expand current Phase 1 scanner scope.

## `archives/BLUEPRINT_REFERENCE/`
Preserved prior blueprint text references.
Contains older Aurora and ISS blueprint-style text files useful for provenance and no-drift comparison.

Use this for:
- comparing prior blueprint doctrine
- recovering wording or old architectural intent
- understanding legacy conceptual layers

## `archives/EXTRACTED_REFERENCE/`
Preserved extracted code/reference assets.
Currently includes runtime cadence and export-contract references.

Use this for:
- code archaeology
- output contract comparison
- legacy runtime pattern recovery

## `archives/LEGACY_SYSTEMS/`
Preserved earlier EA/system attempts and original files.
Includes AFS, EA1, ISSX, and related originals.

Use this for:
- finding reusable design primitives
- checking older module decomposition
- understanding past implementation attempts

## `archives/MAPS/`
Preserved classification/map/code fragments and unclassified historical materials.
Includes backup code and older unclassified folders.

Use this for:
- map recovery
- symbol/classification provenance
- old code reference lookup

# 6. HOW TO USE ARCHIVES SAFELY

## For current scanner work
Use archives only to:
- recover past naming
- recover old module ideas
- recover classification assets
- avoid rediscovering prior mistakes

## For future Aurora research work
Use `archives/Aurora/` to:
- group books by knowledge domain
- identify future extraction candidates
- define later rulecard/research tracks

## What not to do
- do not copy old complexity directly into Phase 1 MT5 work
- do not treat archive text as active contract without blueprint translation
- do not edit archives to make them cleaner
