## Main contradictions and their resolution

### Contradiction A: three layers vs five layers
Older archive canon uses a 3-layer model:
- universe heartbeat
- active surface
- deep shortlist

Newest blueprint uses a 5-layer model:
- open/closed truth
- open-symbol snapshot
- filter
- top-list selection
- deep selective analysis

#### Resolution
Treat the new 5-layer model as an elaborated refinement of the old 3-layer model.

Mapping:
- old Layer 1 -> new Layer 1 plus parts of Layer 2
- old Layer 2 -> new Layer 3 plus Layer 4
- old Layer 3 -> new Layer 5

### Contradiction B: exact old output sections vs new dossier evolution
Older archive canon hard-locked three major symbol sections:
- `[BROKER_SPEC]`
- `[OHLC_HISTORY]`
- `[CALCULATIONS]`

Newest blueprint defines a richer evolving dossier with architecture-owned sections and publication state.

#### Resolution
Use the newer evolving dossier as the internal canonical storage/publication model.
Keep the older three-section schema as a **compatibility export view** if needed, not as the full internal dossier architecture.

### Contradiction C: broker identity from company vs server-only identity
Older docs mention broker/account-company style scoping.
Newest foundation pass explicitly says server-only cleaned identity.

#### Resolution
Use **cleaned account server name only** for file root identity.

### Contradiction D: old flat production folder vs new semantic MT5 structure
Older archive insisted production deployment is flat.
Newest blueprint introduces semantic folders under `Include/ASC`.

#### Resolution
Use a **semantic source tree for maintainability**, but keep final EA include/use patterns simple and compatible with MT5 build expectations.
For the current foundation repo, the flat surface remains a staging build surface.
For the long-term codegen target, use the structured `Include/ASC/...` layout.

### Contradiction E: summary top-5 by bucket vs foundation summary scaffold
Older archive expects ranked bucket leaders.
Newest active foundation blocks ranking for now.

#### Resolution
Foundation summary may be a scaffold or market-state rollup only.
True bucketed top-5 summary remains blocked until Layer 3/4 logic exists.
