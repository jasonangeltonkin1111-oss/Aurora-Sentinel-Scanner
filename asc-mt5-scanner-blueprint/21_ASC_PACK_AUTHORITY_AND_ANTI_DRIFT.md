# ASC Pack Authority and Anti-Drift

## Purpose

This file prevents a future mistake where this deep markdown pack is treated as a parallel constitution competing with the root `blueprint/` folder.

That would create architecture drift instead of reducing it.

## Final authority model

### Root constitutional layer
- `blueprint/`

This remains the primary constitutional source for active ASC system law.

### Deep implementation layer
- `asc-mt5-scanner-blueprint/`

This pack exists to:
- resolve archive contradictions
- deepen implementation detail
- improve code generation quality
- reduce architectural hallucination

It does not exist to replace the root constitutional layer.

## Correct use of this pack

Use this pack when:
- generating code
- resolving archive conflicts
- needing deeper runtime clarification
- needing implementation skeletons, API crosswalks, or acceptance guidance

Do not use this pack to:
- silently override root blueprint law
- preserve broken runtime naming just because it appeared in current code
- revive archive structures that root blueprint has already superseded

## Conflict rule

If this pack and the root `blueprint/` disagree:
1. root `blueprint/` wins
2. this pack must be updated or interpreted as secondary

## Why this matters

Large deep packs often drift into becoming shadow constitutions.
That would be especially dangerous for ASC because the archive already contains multiple architectural eras.

The point of this pack is to reduce ambiguity, not add a new layer of authority confusion.

## Safe interpretation rule

Interpret this pack as:
- deep manual
- implementation bridge
- archive conflict resolver
- codegen support layer

Not as:
- a replacement constitution
- an excuse to ignore active root blueprint law

## Final rule

This pack sharpens the active blueprint.
It does not outrank it.
