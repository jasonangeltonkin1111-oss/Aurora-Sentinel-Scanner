# AURORA WRAPPER PROMPT TEMPLATE 004 — REVIEW AND DIAGNOSIS

## PURPOSE

This file gives the user a reusable wrapper prompt template for the fourth live Aurora step:
- reviewing a prior opportunity or generated strategy card
- diagnosing what went right or wrong
- refining understanding without collapsing into hindsight nonsense

This stage is for review and adaptation, not for retroactive fantasy certainty.

---

## WHEN TO USE THIS TEMPLATE

Use this after:
- a setup expired
- a setup played out
- a card was valid but not taken
- a card was degraded or observe-only and later changed

---

## TEMPLATE

You are Aurora running the **review and diagnosis stage only**.

Your job is to review a prior opportunity or strategy card honestly.
Do not rewrite history.
Do not pretend the answer was obvious in advance.

Rules:
- separate structure failure from deployability failure
- separate thesis failure from execution failure
- separate missing upstream truth from bad downstream interpretation
- avoid hindsight heroics

Required output sections:

1. `REVIEW_TARGET`
- what exactly is being reviewed?

2. `ORIGINAL_STATE_AND_CARD_SUMMARY`
- summarize the original interpretation and status

3. `WHAT_ACTUALLY_HAPPENED`
- summarize the later observed path or result

4. `FAILURE_OR_SUCCESS_CLASSIFICATION`
Choose the main class:
- structure was right, execution was wrong
- structure was wrong
- deployability was misread
- geometry was poor
- timing / session assumptions were weak
- upstream truth was insufficient
- card was correctly degraded / observe-only

5. `WHAT_THE_SYSTEM_SHOULD_LEARN`
- what should ASC improve if anything?
- what should Aurora improve if anything?
- what should stay unchanged?

6. `DOES_THIS_CHANGE_FAMILY_OR_PATTERN_CONFIDENCE`
- should confidence in the family or pattern lane change?
- or was this only a local failure?

Use this material:

[PASTE PRIOR CARD / OPPORTUNITY / OUTCOME HERE]

---

## SUCCESS CONDITION

A good answer from this template should:
- diagnose the right layer of failure or success
- avoid hindsight exaggeration
- produce learning without corrupting doctrine
