# AURORA STRATEGY FAMILY REGISTRY

## PURPOSE

This file is the first live strategy-family registry for Aurora Blueprint.

It exists to preserve and organize the actual strategy families and strategy classes referenced across the Aurora vault-style material without blindly importing the full governing/gating language that originally surrounded them.

This file is therefore:
- a family-preservation file
- a doctrine-placement file
- a strategy-organization file

This file is **not**:
- a re-import of the full vault constitution
- a copy of suppression law
- a copy of execution governance
- a proof that every family is already active, complete, or deployable in chart-only Wave 1 work

The goal here is simple:
- do not lose academically grounded strategy families
- place each family where it belongs
- separate what is ready for direct Aurora use from what must remain conditional or deferred

---

# 1. REGISTRY LOGIC

This registry organizes strategies into four placement classes:

## Class A — CORE AURORA FAMILIES
Families that directly inherit from the current Wave 1 state/surface doctrine and can become the first real Aurora strategy families.

## Class B — CONDITIONAL / EXTENDED FAMILIES
Families that are real and worth preserving, but require extra context, session framing, instrument grouping, or stronger testing doctrine before they can become clean Aurora core families.

## Class C — STRUCTURE-PRESERVABLE BUT NOT CHART-ONLY NATIVE
Families that are academically real and should remain in Aurora memory, but do not belong in chart-only core admissibility without additional data surfaces.

## Class D — FUTURE DERIVATIVE OR HYBRID LAYERS
Families that may later be assembled from the registry once the base family logic is stable.

---

# 2. CLASS A — CORE AURORA FAMILIES

These are the highest-priority strategy families because they fit the current Wave 1 doctrine stack most naturally.

## A1. Trend Continuation Family

### Canonical idea
Participate when the market is already in a directional discovery / expansion condition and the continuation structure remains stronger than competing reversal or balance interpretations.

### Native habitat
- Trend Expansion
- strong Directional Discovery
- clean post-acceptance continuation

### Typical structural signatures
- accepted break from prior balance
- directional follow-through
- shallow or orderly pullback behavior
- responsive opposition weaker than initiative continuation

### Why it belongs in Aurora now
This is the cleanest direct descendant of the current Wave 1 market-state canon.

### Closely related source labels
- trend family
- TS momentum
- channel break with acceptance
- directional continuation

### Main anti-confusions
Do not collapse this with:
- breakout from compression before acceptance
- weak attempted direction
- failed break conditions

---

## A2. Trend Pullback / Pullback Continuation Family

### Canonical idea
Participate when a directional market remains structurally intact, but the opportunity appears through temporary retracement rather than immediate expansion print.

### Native habitat
- Trend Pullback
- strong Directional Discovery that has not degraded

### Typical structural signatures
- prior directional discovery already established
- retracement that does not fully destroy the prior directional structure
- continuation still dominates over reversal and return-to-balance logic

### Why it belongs in Aurora now
Wave 1 already contains the structural language needed to host this family, even if some details remain provisional.

### Closely related source labels
- pullback continuation
- trend resumption
- MA-filtered trend family

### Main anti-confusions
Do not activate this when:
- pullback is actually a failed state
- balance is re-forming
- overlap / compression is growing instead of resolving

---

## A3. Breakout / Compression Release Family

### Canonical idea
Participate when a compressed or balanced market resolves into directional expansion with real structural commitment rather than cheap excursion.

### Native habitat
- Compression resolving into Breakout / Expansion Transition
- Balance / Range resolving into Directional Discovery

### Typical structural signatures
- prior compression or balance clearly visible
- break beyond reference structure
- acceptance or persistence beyond the break
- continuation trust stronger than fake-break / trap risk

### Why it belongs in Aurora now
This is one of the main family types already implied by the Wave 1 stack and by the earlier starter-pack logic.

### Closely related source labels
- breakout family
- channel break with acceptance
- compression release
- range break with hold

### Main anti-confusions
Do not collapse this with:
- one-bar spike
- wick-only excursion
- unresolved false commitment
- low-trust expansion inside overlap

---

## A4. Failed Break / Trap Reversal Family

### Canonical idea
Participate when an attempted directional move loses acceptance, reclaims prior structure, and reversal / return-to-structure logic becomes dominant.

### Native habitat
- Failed State / Trap
- Failed Break / Trap State
- false commitment reverting back through initiating structure

### Typical structural signatures
- visible break attempt
- inability to hold beyond the break
- reclaim of prior structure
- responsive logic stronger than continuation logic

### Why it belongs in Aurora now
This family is already deeply supported by the current Wave 1 doctrine stack.

### Closely related source labels
- trap family
- failed-state family
- failed break reversal
- reclaim reversal

### Main anti-confusions
Do not activate this when:
- the break is still unresolved
- rejection is not yet structurally clear
- multiple alternating traps make the environment unstable

---

## A5. Balance Rotation / Range Mean-Reversion Family

### Canonical idea
Participate when a stable balance / range condition supports return-to-value or opposite-boundary rotation more strongly than breakout continuation.

### Native habitat
- Balance / Range
- Range Extremes
- rotational accepted trade

### Typical structural signatures
- anchor stability
- repeated rejection at boundaries
- internal rotation stronger than directional migration
- no unresolved breakout attempt dominating the structure

### Why it belongs in Aurora now
This is one of the fundamental family types already present in the archive logic and the Wave 1 doctrine.

### Closely related source labels
- range family
- mean-reversion family
- balance rotation
- fade the extremes

### Main anti-confusions
Do not activate this when:
- balance is unstable
- breakout acceptance is emerging
- overlap expansion is widening into chaos rather than stable rotation

# 3. CLASS B — CONDITIONAL / EXTENDED FAMILIES

These are real and important, but should not yet be treated as clean first-wave core families without stronger supporting doctrine.

## B1. Intraday Momentum Family

### Canonical idea
Short-horizon momentum logic defined partly by session structure, intraday liquidity cycles, and time-of-day context.

### Why conditional
This family is real, but needs explicit session definitions and testing discipline beyond a single screenshot or generic chart view.

### Future destination
- Strategy Family Registry (extended section)
- later session-specific execution notes

---

## B2. Opening Range Breakout (ORB) Variants

### Canonical idea
Trade breakout or post-open directional resolution relative to a defined opening range.

### Why conditional
This is academically and practically real, but requires explicit timebox definition, session framing, and more exact operational constraints.

### Future destination
- Strategy Family Registry
- later Pattern Atlas / session module support

---

## B3. Cross-Sectional Momentum Family

### Canonical idea
Relative-strength / relative-weakness logic across instruments rather than isolated single-chart structure alone.

### Why conditional
This is a valid strategy family, but Aurora would need stronger cross-asset grouping and ranking doctrine before treating it as active.

### Future destination
- Strategy Family Registry
- later research/testing support

---

## B4. Calendar / Seasonal Effects Family

### Canonical idea
Participation based partly on repeatable calendar or time-window effects.

### Why conditional
This is real but not native to the current chart-only Wave 1 doctrine, and it requires stronger research-method protection against overfitting.

### Future destination
- Strategy Family Registry
- Research / Testing Method

# 4. CLASS C — STRUCTURE-PRESERVABLE BUT NOT CHART-ONLY NATIVE

These families are worth preserving because they are academically real, but they do not belong in chart-only core Aurora without additional data surfaces or research layers.

## C1. Pairs / Statistical Arbitrage Family

### Canonical idea
Exploit relative mispricing, spread behavior, or co-movement distortions across linked instruments.

### Why preserved
This is too important to lose, but it is not a clean direct child of the current single-chart Wave 1 state language.

### Why not core now
Requires spread construction, pair logic, or multi-series statistical framing.

### Future destination
- Strategy Family Registry (preserved / deferred)
- Research / Testing Method

---

## C2. Order-Book / Microstructure Tactics Family

### Canonical idea
Execution or directional tactics driven by live order-book, queue, or hidden-liquidity behavior.

### Why preserved
These are real tactics, but not chart-only native.

### Why not core now
Require data Aurora does not yet treat as core admissibility input.

### Future destination
- preserved as non-core strategy class
- possible later advanced execution layer

---

## C3. Options Overlay / Volatility Harvest Families

### Canonical idea
Systematic options-based or volatility-premium strategies such as buy-write / put-write / VRP-like overlays.

### Why preserved
Academically real, materially important, but outside current chart-only Wave 1 core.

### Why not core now
Require options surface, contract logic, and risk framework not yet present.

### Future destination
- preserved non-core registry memory
- later volatility / risk and derivatives layer

---

## C4. HFT / Latency / Ultra-Short-Horizon Tactics

### Canonical idea
Strategies that depend on queue position, latency, microsecond reactions, or ultra-short-lived order-flow asymmetry.

### Why preserved
Real, but architecturally separate from current Aurora doctrine.

### Why not core now
They are not compatible with the present chart-only doctrine surface.

### Future destination
- preserved as excluded strategy class
- not part of immediate Aurora core

---

# 5. CLASS D — FUTURE DERIVATIVE OR HYBRID LAYERS

These are not separate immediate registry families yet, but likely future structured combinations.

## D1. Breakout Continuation vs Failed Break Split Framework
Use the same parent breakout state and explicitly branch it into:
- true continuation family
- failed-break reversal family

This should later become a major Aurora design strength.

## D2. Balanced-to-Expansion Transition Family Pack
Use balance / compression doctrine to split future active subfamilies such as:
- early release continuation
- false release rejection
- post-break pullback continuation

## D3. Session-Conditioned Family Pack
Once session doctrine is stronger, families like:
- London momentum
- ORB variants
- post-open reversal families
may become a formal hybrid cluster.
