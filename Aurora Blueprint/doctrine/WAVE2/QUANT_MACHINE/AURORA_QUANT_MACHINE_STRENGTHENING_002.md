# 2. QUANT / MACHINE STRENGTHENING ADDITIONS

## 2.1 Finance-ML-is-not-free-form rule

### Strengthening statement
Machine learning in finance should be treated as a domain-constrained research practice, not as unrestricted pattern hunting.

### Why this is justified
Financial Machine Learning frames ML in finance around domain-specific challenges and the continuing value of economic structure.

### Canonical implication
Aurora’s machine lane should preserve the rule that financial ML remains subordinate to:
- market structure
- family logic
- evidence discipline
- implementation reality

---

## 2.2 Large-information-set rule

### Strengthening statement
The machine lane should remain open to using wider information sets than human discretionary reasoning can comfortably process, provided that the added information survives disciplined testing.

### Why this is justified
Financial Machine Learning explicitly motivates ML through large information sets and ambiguous functional forms.

### Canonical implication
Aurora should preserve a future path for broader feature sets, but only under strong testing and comparison discipline.

---

## 2.3 Complexity-wedge rule

### Strengthening statement
Complexity should be viewed as a wedge with both upside and danger; bigger models are not automatically better.

### Why this is justified
The ML survey devotes major attention to model complexity, comparative analysis, and the conditions under which complex models help or fail.

### Canonical implication
Aurora’s machine lane should preserve explicit questions such as:
- what simple benchmark is being beaten?
- what robustness survives?
- what extra fragility is introduced?
- what explanatory value is lost?

---

## 2.4 Prediction-to-portfolio continuity rule

### Strengthening statement
The machine lane should not stop at return prediction; it should preserve continuity into portfolio choice, trading costs, and deployment consequences.

### Why this is justified
Financial Machine Learning extends from return prediction into risk-return tradeoffs, optimal portfolios, trading costs, and reinforcement-learning context.

### Canonical implication
Aurora’s machine lane should later preserve bridges from signal prediction into:
- weighting
- ranking
- cost-aware deployment
- suppression under fragile conditions

---

## 2.5 Expected-return anchoring rule

### Strengthening statement
Machine predictions should remain anchored to explicit expected-return reasoning where possible rather than becoming detached scores with no economic interpretation.

### Why this is justified
Expected Returns on Major Asset Classes provides a clear expected-return and premia framing that can constrain machine outputs.

### Canonical implication
Aurora’s machine lane should preserve a path where ML outputs can later be related to expected-return surfaces, premia context, and scenario pressure.
