# 3. QUANT-LANE STRENGTHENING ADDITIONS

## 3.1 Every serious family should have a measurable translation path

### Strengthening statement
A serious family should not remain only a descriptive chart narrative.
It should be capable of later translation into measurable rule candidates.

### Why this is justified
The quantitative sources are strongest when they show how to turn an idea into:
- variables
- explicit conditions
- testable logic
- comparable strategy forms

### Canonical implication
Every core family should later be able to answer:
- what measurable variables define this family?
- what structural conditions need proxy fields?
- what evidence would strengthen or weaken the family?

### What this changes
This strengthens the family-card and evidence-note architecture by requiring later rule-translation fields rather than leaving the cards purely descriptive.

---

## 3.2 Backtesting belongs near the family layer, not only near final implementation

### Strengthening statement
Backtesting should be treated as a family-maturity function, not merely a last-stage coding exercise.

### Why this is justified
The quantitative sources place testing, comparison, and refinement near the center of strategy development.

### Canonical implication
Family development should eventually include:
- measurable family hypotheses
- historical testing paths
- evidence questions
- anti-overfitting checks

### What this changes
This strengthens the Strategy Testing / Evidence Layer and justifies later family evidence notes as a core lane rather than optional add-ons.

## 3.5 Machine learning is a later refinement layer, not a shortcut around structure

### Strengthening statement
Machine learning should later help refine, rank, or compare strategy forms, but it should not replace family logic, habitat logic, or anti-habitat logic.

### Why this is justified
The quant/Python sources preserve ML as one extension path within strategy development, not as a reason to skip structural interpretation.

### Canonical implication
ML should later support:
- feature selection and testing
- classification refinement
- ranking or comparison
- later pattern/family evidence work

It should not erase:
- family meaning
- pattern meaning
- state/surface dependence

### What this changes
This keeps the machine lane aligned with Aurora doctrine instead of turning it into detached model-chasing.
