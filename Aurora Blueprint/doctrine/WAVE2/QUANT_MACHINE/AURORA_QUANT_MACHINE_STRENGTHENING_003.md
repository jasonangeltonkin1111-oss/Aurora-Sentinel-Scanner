# 2. QUANT / MACHINE STRENGTHENING ADDITIONS

## 2.1 Finance-ML-is-special rule

### Strengthening statement
Financial ML should be treated as a domain-specific workflow with unique traps, not as generic ML pasted onto market data.

### Why this is justified
Advances in Financial Machine Learning explicitly frames financial ML as a distinct subject because of finance-specific noise, overfitting, and validation problems.

### Canonical implication
Aurora’s machine lane should preserve domain-specific caution whenever ML methods are imported.

---

## 2.2 Workflow-over-lucky-strike rule

### Strengthening statement
The machine lane should be built as a production process, not as isolated model hunts.

### Why this is justified
Advances in Financial Machine Learning contrasts isolated strategy hunting with a meta-strategy / research-factory workflow.

### Canonical implication
Aurora’s machine lane should preserve role separation and process structure rather than hero-model culture.
