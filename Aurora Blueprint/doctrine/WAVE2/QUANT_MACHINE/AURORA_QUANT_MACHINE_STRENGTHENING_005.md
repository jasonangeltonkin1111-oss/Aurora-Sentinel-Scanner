# 2. QUANT / MACHINE STRENGTHENING ADDITIONS

## 2.1 Process-first-quant rule

### Strengthening statement
Quant/machine work should be treated as a staged process from strategy discovery through backtest, implementation, scaling, and risk management.

### Why this is justified
Chan 2008 explicitly frames quantitative trading as building and running an algorithmic trading business, not merely discovering models. fileciteturn117file1

### Canonical implication
Aurora’s machine lane should preserve stronger process structure rather than collapsing everything into model selection.

---

## 2.2 Mechanics-aware-machine rule

### Strengthening statement
The machine lane should remain aware of electronic market structure, order types, price impact, and order-flow mechanics.

### Why this is justified
Peng Liu devotes early chapters to electronic markets, order matching, LOB, price impact, and order flow before later model chapters. fileciteturn117file0

### Canonical implication
Wave 2 should preserve stronger market-mechanics awareness in quant/machine design.

---

## 2.3 Bounded-ML-expansion rule

### Strengthening statement
Machine learning should be used as a bounded enlargement of the modeling space in finance rather than as a license to ignore structure.

### Why this is justified
Kelly and Xiu explicitly motivate ML through large information sets and ambiguous functional forms while preserving the role of economic structure. fileciteturn117file3

### Canonical implication
Aurora’s machine lane should preserve bounded ML expansion rather than generic ML maximalism.

---

## 2.4 Stat-arb-serious-branch rule

### Strengthening statement
Statistical arbitrage should be preserved as a serious family/quant branch with calibration, factor control, and market-impact awareness.

### Why this is justified
Pole’s contents explicitly include pair identification, spread margins, event analysis, market-factor exposure, market impact, and calibration. fileciteturn117file4

### Canonical implication
Wave 2 should preserve stat-arb as a real calibrated branch instead of flattening it into generic mean reversion.

---

## 2.5 Complexity-needs-rationale rule

### Strengthening statement
Complexity in quant systems should be justified by robustness or information value rather than by sophistication theater.

### Why this is justified
Chan 2013 explicitly emphasizes simple and linear strategies as an antidote to overfitting and data-snooping. fileciteturn117file2

### Canonical implication
Aurora’s machine lane should preserve stronger suspicion toward gratuitous complexity.
