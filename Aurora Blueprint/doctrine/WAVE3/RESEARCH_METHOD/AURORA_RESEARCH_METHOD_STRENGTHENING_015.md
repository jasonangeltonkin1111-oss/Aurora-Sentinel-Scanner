# 2. RESEARCH-METHOD STRENGTHENING ADDITIONS

## 2.1 Event-construction rule

### Strengthening statement
Observation construction should be treated as part of the hypothesis design rather than as a neutral preprocessing step.

### Why this is justified
AFML begins its practical workflow with financial data structures, bars, and sampling features instead of assuming that naive clock-time slicing is good enough.

### Canonical implication
Wave 3 should preserve stronger scrutiny of how observations are sampled before any model is praised.

---

## 2.2 Label-design rule

### Strengthening statement
Labels should reflect economically meaningful event outcomes rather than defaulting to naive fixed-horizon classification.

### Why this is justified
AFML explicitly elevates labeling, triple-barrier logic, side-vs-size separation, and meta-labeling into load-bearing financial-ML concerns.

### Canonical implication
Wave 3 should preserve label architecture as a first-class research decision.

---

## 2.3 Uniqueness-aware-weighting rule

### Strengthening statement
Sample influence should be adjusted when outcomes overlap and multiple labels reuse the same realized return path.

### Why this is justified
AFML explicitly develops concurrent-label counts, average uniqueness, return attribution, and time decay because financial samples are not IID.

### Canonical implication
Wave 3 should preserve stronger pressure against pretending dependent samples are independent evidence.

---

## 2.4 Memory-preserving-stationarity rule

### Strengthening statement
Stationarity transformations should preserve as much useful memory as possible instead of automatically deleting predictive structure.

### Why this is justified
AFML explicitly presents fractional differentiation as a solution to the stationarity-versus-memory dilemma.

### Canonical implication
Wave 3 should preserve preprocessing skepticism whenever invariance is achieved by information destruction.

---

## 2.5 Purged-validation rule

### Strengthening statement
Cross-validation in finance should be purged and embargoed when temporal dependence or outcome overlap could leak information across folds.

### Why this is justified
AFML explicitly argues that naive K-fold CV fails in finance and develops purged K-fold and embargo logic as the corrective discipline.

### Canonical implication
Wave 3 should preserve purged validation as the default posture whenever financial dependence structure makes ordinary CV unsafe.

---

## 2.6 Backtest-failure rule

### Strengthening statement
Research validity should include failure probability, structural-break sensitivity, and overfit defenses rather than treating backtest quality as self-validating.

### Why this is justified
AFML explicitly ties backtest danger, combinatorial purged cross-validation, strategy-risk analysis, structural breaks, and microstructural awareness into one workflow.

### Canonical implication
Wave 3 should preserve stronger pressure against declaring victory from historical fit alone.

---

## 2.7 Tested-state-awareness rule

### Strengthening statement
State/regime-aware evaluation is legitimate when evidence supports distinct dynamic structures, but switching stories should be tested rather than assumed.

### Why this is justified
The Markov-switching note formalizes hidden-state switching in mean and variance together with estimation and hypothesis testing for switching structure.

### Canonical implication
Wave 3 should preserve room for regime-sensitive evaluation while requiring explicit test pressure.
