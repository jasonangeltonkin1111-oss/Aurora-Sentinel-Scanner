## WHY THIS LAYOUT WAS CHOSEN

The wrapper package needed to reduce file count without becoming either:
- thousands of tiny wrapper fragments
- one giant monolithic wrapper dump

The chosen structure uses:
- one tiny kernel/router
- one settings file
- one routing/file-map file
- five major canonical packs
- one maintenance guide

This keeps the hot path small while allowing whole-pack replacement when source areas evolve materially.
