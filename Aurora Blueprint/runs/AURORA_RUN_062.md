## REPO-TRUTH JUDGMENT

### What already existed implicitly
Family competition truth already existed in scattered form across:
- the wrapper object model as an object placeholder
- the workflow packet as a combined family/pattern stage
- the family system map as a stated system layer
- family cards and family files through habitats, anti-habitats, competitors, and rejection conditions
- the C-04 / G4 packet and worked example through explicit competitor reasoning
- the prompt template through family-comparison questioning

### What was missing explicitly
What was missing was one canonical Aurora surface defining:
- candidate-family generation
- non-numeric competition dimensions
- elimination law
- ambiguity statuses
- exact object shape
- honest invalid-input and deferred-classification behavior

### Duplicate-truth risk that was avoided
The pass intentionally did not pull habitat doctrine, anti-habitat doctrine, card logic, pattern doctrine, or opportunity rules out of their owning files.
The new engine consumes those surfaces rather than replacing them.

## WHAT MISSING LAYER WAS CLOSED

This run closed the missing dedicated family competition layer by adding:
- canonical family competition protocol truth
- canonical family competition object schema truth
- one worked example showing contested-primary behavior with explicit rejection reasons and preserved ambiguity

Aurora can now represent:
- clear primary
- contested primary
- multiple live families
- deferred classification
- no valid family
- invalid competition input

without using hidden score logic or forcing a family winner.

## WHAT SURROUNDING SURFACES WERE TIGHTENED

- the enum registry now has a dedicated family-competition-status enum set
- the wrapper workflow now states family competition before pattern competition explicitly
- the wrapper object model now gives `AURORA_FAMILY_COMPETITION_OBJECT` a cleaner canonical field set and cross-references its protocol/schema
- the family/pattern prompt template now asks for a family competition status before any pattern judgment
- the opportunity inventory protocol now treats family competition as a structured upstream input instead of an implied ranking blob

- the family competition layer is now formalized, but later worked examples can still deepen real ASC-backed cases
- family doctrine remains enrichable inside the family files and cards without changing the engine law
