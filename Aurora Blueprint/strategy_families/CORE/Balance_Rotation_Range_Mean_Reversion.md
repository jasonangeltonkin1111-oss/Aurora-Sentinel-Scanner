# BALANCE ROTATION / RANGE MEAN-REVERSION

## PURPOSE

This file defines the core Aurora Balance Rotation / Range Mean-Reversion family.

It is written as a family-definition file, not as a final trading ruleset.

Its job is to make the family:
- structurally clear
- habitat-clear
- anti-confusion clear
- multi-surface aware for later wrapper / EA use

---

# 1. FAMILY NAME

## Canonical name
Balance Rotation / Range Mean-Reversion

## Closely related labels
- range family
- mean-reversion family
- balance rotation
- fade the extremes

---

# 2. CANONICAL IDEA

Participate when a stable balance / range condition supports return-to-value or opposite-boundary rotation more strongly than breakout continuation.

This family is not about fading all movement.
It is about using balance stability and rejection behavior where rotational logic remains stronger than directional migration.

---

# 3. NATIVE HABITAT

Primary native habitats:
- Balance / Range
- Range Extremes
- rotational accepted trade

This family belongs where:
- anchor stability is present
- repeated rejection at boundaries is visible
- internal rotation dominates over directional migration
- breakout acceptance is not yet taking control

---

# 4. ANTI-HABITAT

This family should not be treated as native in:
- unstable balance
- clear breakout acceptance
- widening chaotic overlap with no clean rotation logic
- transition to strong directional discovery
- failed break environment where reversal is no longer merely rotational

---

# 5. STRUCTURAL SIGNATURES

Typical structural signatures include:
- stable accepted area
- repeated rejection at or near meaningful boundaries
- rotational behavior stronger than sustained directional hold
- return-to-value logic stronger than continuation logic

---

# 6. MAIN COMPETING INTERPRETATIONS

Balance Rotation / Range Mean-Reversion must be distinguished from:
- Breakout / Compression Release
- Failed Break / Trap Reversal
- Trend Continuation
- Trend Pullback / Pullback Continuation

The family is only valid when rotation remains stronger than migration.

---

# 7. MAIN ANTI-CONFUSIONS

Do not activate this family when:
- balance is unstable
- breakout acceptance is emerging
- overlap expansion is widening into chaos rather than stable rotation
- directional authority is becoming stronger than rotational rejection logic
