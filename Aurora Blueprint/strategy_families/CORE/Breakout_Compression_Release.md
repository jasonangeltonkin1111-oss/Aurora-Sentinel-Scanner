# BREAKOUT / COMPRESSION RELEASE

## PURPOSE

This file defines the core Aurora Breakout / Compression Release family.

It is written as a family-definition file, not as a final trading ruleset.

Its job is to make the family:
- structurally clear
- habitat-clear
- anti-confusion clear
- multi-surface aware for later wrapper / EA use

---

# 1. FAMILY NAME

## Canonical name
Breakout / Compression Release

## Closely related labels
- breakout family
- compression release
- range break with hold
- channel break with acceptance

---

# 2. CANONICAL IDEA

Participate when a compressed or balanced market resolves into directional expansion with real structural commitment rather than cheap excursion.

This family is not about every visible boundary break.
It is about release that survives as meaningful expansion.

---

# 3. NATIVE HABITAT

Primary native habitats:
- Compression resolving into Breakout / Expansion Transition
- Balance / Range resolving into Directional Discovery

This family belongs where:
- prior compression or balance is clear
- release beyond structure becomes more than momentary excursion
- continuation trust becomes stronger than fake-break / trap risk

---

# 4. ANTI-HABITAT

This family should not be treated as native in:
- wick-only excursion without acceptance
- unstable overlap with no clean release
- obvious trap / failed break environments
- balance that is still holding cleanly
- late exhausted extension rather than fresh release

---

# 5. STRUCTURAL SIGNATURES

Typical structural signatures include:
- prior compression or balance clearly visible
- break beyond reference structure
- persistence or acceptance beyond the break
- continuation trust stronger than reversal/trap logic
- reduced evidence that the move is only cheap excursion

---

# 6. MAIN COMPETING INTERPRETATIONS

Breakout / Compression Release must be distinguished from:
- Trend Continuation
- Failed Break / Trap Reversal
- Balance Rotation / Range Mean-Reversion
- low-trust directional attempt with poor hold quality

The family is only valid when release dominates over rejection.

---

# 7. MAIN ANTI-CONFUSIONS

Do not collapse this family with:
- one-bar spike
- wick-only excursion
- unresolved false commitment
- low-trust expansion inside overlap
- continuation after the release is already mature and exhausted
