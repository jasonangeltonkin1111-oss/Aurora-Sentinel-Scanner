# 5. FUTURE BOOK ENRICHMENT RULE

When future books are processed, the strategy layer should normally be enriched in this order:

1. update family-definition files if the family itself becomes more precise
2. update family cards with:
   - better habitat definitions
   - better invalidation logic
   - better rejection conditions
   - better data-surface dependencies
   - better testing/research needs
3. update the family matrix if readiness/classification changes
4. update this sweep log only if the overall strategy-layer structure changes materially

Do not create redundant side files if a clear card or family file can absorb the new knowledge safely.
