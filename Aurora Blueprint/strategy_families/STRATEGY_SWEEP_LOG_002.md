---

# 5. FUTURE ENRICHMENT LAW

As more books are processed, the strategy layer should normally be enriched in this order:

1. doctrine if a new source materially changes family meaning
2. family-definition files if family boundaries sharpen
3. family cards if habitat, invalidation, or rejection logic improves
4. pattern atlas if new child groups become justified
5. pattern cards if pattern meanings or misreads sharpen
6. matrix / system map only if the overall architecture changes materially

Do not create redundant side files when an existing family card or pattern card can safely absorb the new knowledge.
