# FAMILY NAME
Failed Break / Trap Reversal

BUILD_READY

# CANONICAL THESIS
Participate when an attempted directional move loses acceptance, reclaims prior structure, and reversal or return-to-structure logic becomes dominant.

# PRIMARY MARKET HABITAT
- Failed State / Trap
- Failed Break / Trap State
- false commitment reverting back through initiating structure

# PRIMARY STATE / SURFACE DEPENDENCIES
## State dependencies
- Failed Break / Trap State
- Transitional / Unstable State when failure is becoming clearer than continuation

## Surface dependencies
- Return-to-Value / Responsive Surface often supportive
- Structurally Valid but Low-Trust Surface may precede stronger reversal confirmation

# STRUCTURAL ENTRY LOGIC (HIGH LEVEL ONLY)
At a high level, the family becomes plausible when:
- a visible break attempt occurred
- continuation could not hold
- rejection and reclaim of prior structure become stronger than continuation logic
- responsive logic dominates over renewed directional authority

# STRUCTURAL INVALIDATION LOGIC (HIGH LEVEL ONLY)
At a high level, the family weakens or invalidates when:
- the break is still unresolved rather than failed
- renewed acceptance beyond the original break restores continuation authority
- trap-on-trap instability prevents a clean failure hierarchy

# MAIN FAMILY COMPETITORS
- Breakout / Compression Release
- Trend Continuation
- Balance Rotation / Range Mean-Reversion

# FAMILY REJECTION CONDITIONS
Reject or suppress the family when:
- rejection is not yet structurally clear
- the move is only ordinary pullback rather than actual failure
- continuation logic is still stronger than reclaim/reversal logic
- environment is too unstable to identify clean failure structure

# FUTURE PATTERN CHILDREN

- trap signatures
- false release rejection patterns
- return-through-initiating-structure patterns

# FUTURE TESTING / RESEARCH NEEDS

- role of session/open conditions in trap frequency and quality
- effect of live spread/liquidity conditions on reclaim trust
