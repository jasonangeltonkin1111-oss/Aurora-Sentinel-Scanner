# AURORA STARTER FAMILY BUILD

## PURPOSE

This file defines the first strategy-family build pack Aurora should use.

It exists to turn the family registry into an actionable and organized family-construction layer.

This is the bridge between:
- doctrine
- family preservation
- future wrapper / EA design

This file is not the final implementation of each family.
It is the organized starting build map.

---

# 1. FIRST BUILD PACK

The first Aurora family pack should contain exactly these five core families:

1. Trend Continuation
2. Trend Pullback / Pullback Continuation
3. Breakout / Compression Release
4. Failed Break / Trap Reversal
5. Balance Rotation / Range Mean-Reversion

Why these first:
- they map directly to current Wave 1 doctrine
- together they cover the main structural habitats
- they are the least dependent on non-chart-only infrastructure
- they preserve the most important strategy families from the existing knowledge set

---

# 2. STRUCTURAL COVERAGE OF THE FIRST BUILD PACK

## Trend habitats covered
- Trend Continuation
- Trend Pullback / Pullback Continuation

## Transition / expansion habitats covered
- Breakout / Compression Release

## Failure / reversal habitats covered
- Failed Break / Trap Reversal

## Balance / rotational habitats covered
- Balance Rotation / Range Mean-Reversion

Together this gives Aurora a broad but still manageable first family architecture.

---

# 3. WHY THIS FILE EXISTS FOR FUTURE GPT WRAPPER / EA WORK

A future GPT wrapper or EA should not have to infer the first strategy set from scattered doctrine.

This file makes the first pack explicit.

That means future build layers can use this file to answer:
- which families exist first
- why they exist first
- which habitat each covers
- which file to read next for that family
