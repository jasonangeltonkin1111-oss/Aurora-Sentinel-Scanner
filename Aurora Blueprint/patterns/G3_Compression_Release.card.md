# PATTERN ID
G3

# PATTERN NAME
Compression Release

# STATUS
APPROVED_GROUP

# PARENT FAMILY / FAMILIES
- Breakout / Compression Release

# CANONICAL THESIS
This pattern group captures structures where visible compression or contained balance resolves into expansion with enough persistence that release is becoming meaningful rather than remaining trapped inside unresolved overlap.

# PRIMARY MARKET HABITAT
- Compression resolving into expansion
- balance-to-break transition

# PRIMARY STATE / SURFACE DEPENDENCIES
## State dependencies
- transition from Balance / Compression toward Directional Discovery

## Surface dependencies
- Discovery-with-Trust Surface preferred
- Structurally Valid but Low-Trust Surface may suppress or downgrade the pattern

# REQUIRED DATA SURFACES
- S1_CHART_STRUCTURE

# HIGH-VALUE SUPPORTING DATA SURFACES
- S2_ASC_INTERNAL_FEATURES
- S3_LIVE_EXECUTION_MICROSTRUCTURE
- S6_CALENDAR_SESSION_TIMEBOX

# STRUCTURAL MEANING
The meaning of this pattern is that containment is giving way to directional release in a way that may justify breakout-family interpretation.

# STRUCTURAL INVALIDATION / FAILURE LOGIC
At a high level, the pattern weakens or fails when:
- expansion cannot persist beyond the compressed structure
- the release immediately degrades into false commitment
- trap or failed-break logic overtakes continuation logic

# MAIN MISREADS / COMPETING INTERPRETATIONS
- cheap excursion mistaken for release
- failed break / reclaim
- unstable overlap mistaken for compression
- already-mature trend continuation mistaken for fresh release

# PATTERN REJECTION CONDITIONS
Reject or suppress the pattern when:
- containment is not actually clear beforehand
- release is not persisting meaningfully
- environment remains dominated by unstable overlap or low trust

# FUTURE TESTING / RESEARCH NEEDS
- compression quality studies
- release persistence diagnostics
- relation between ASC pre-selection and release quality
- session-conditioned breakout behavior
- live spread / liquidity effects on release deployability

# FUTURE ENRICHMENT PATH
Future books and extractions may add:
- narrower compression taxonomies
- stronger release-vs-fake-release distinctions
- better breakout trust filters
- session-specific release variants
