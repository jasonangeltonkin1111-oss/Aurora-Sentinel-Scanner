# AURORA WRAPPER PATTERN VAULT

## Pattern-layer law

Pattern competition is downstream of family competition.
Patterns are local structural roles inside Aurora families, not detached chart labels.
The pattern engine must consume the surviving family set and must not silently reopen family selection.

## Pattern competition rules

- family-first law
- no forced pattern winner
- no numeric-confidence scoring
- preserve missingness and ambiguity
- compare candidates by parent-family compatibility, structural role fit, anti-confusion pressure, state alignment, regime compatibility, contradiction load, and dependence on missing surfaces

Possible outcomes include clear pattern primary, contested pattern primary, multiple live patterns, deferred classification, no valid pattern, and invalid input.

## Current family-to-pattern map

### G1 — Accepted Break and Hold
Primary role: confirms successful acceptance outside prior containment and fits best under breakout/compression-release logic.
Main confusion to avoid: treating a failed excursion as successful acceptance.

### G2 — Orderly Continuation Pullback
Primary role: organizes constructive retracement inside an ongoing directional trend, especially under pullback-continuation logic.
Main confusion to avoid: mistaking messy reversal or exhaustion for an orderly continuation pullback.

### G3 — Compression Release
Primary role: local breakout-release expression inside C-03 structure.
Main confusion to avoid: using it when the release is not materially compressed or when expansion never truly confirms.

### G4 — Failed Break / Reclaim
Primary role: local failed-break recovery and reclaim expression inside C-04 structure.
Main confusion to avoid: misclassifying unresolved breakout drift as a confirmed reclaim.

### G5 — Range Rejection / Return to Value
Primary role: rejection from range edge or failed excursion back toward balance/value under C-05 logic.
Main confusion to avoid: using it when price is actually leaving balance into real trend or breakout behavior.
