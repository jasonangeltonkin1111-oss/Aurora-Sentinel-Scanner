# AURORA WRAPPER FAMILY VAULT

## Family-layer law

Aurora is family-first.
Family competition sits after context interpretation and before pattern competition, opportunity classification, and generated-card decisions.
The family engine consumes doctrine from family source files; it does not replace them with a generic abstraction.

## Family competition rules

- no forced family winner
- no numeric-confidence scoring
- preserve missingness and defer classification when inputs are too thin
- include plausible competitors instead of over-collapsing routing
- compare candidates by habitat fit, anti-habitat conflict, state alignment, regime compatibility, contradiction load, and dependence on missing surfaces

Possible outcomes include clear primary, contested primary, multiple live families, deferred classification, no valid family, and invalid input.

## Core family system

### C-01 — Trend Continuation
Canonical idea: continuation of an already established directional state.
Native habitat: clean directional structure with ongoing participation and low evidence of exhaustion.
Anti-habitat: late, overextended, failure-prone, or balance-dominant conditions.
Main competitors: pullback continuation when retracement dominates, breakout when compression-release posture dominates, failed break when continuation evidence breaks down.

### C-02 — Trend Pullback / Pullback Continuation
Canonical idea: a directional trend resumes through a constructive retracement rather than a fresh base breakout.
Native habitat: established directional bias with orderly pullback behavior and restoration signals.
Anti-habitat: chaotic reversal structure, unresolved balance, or conditions where pullback is actually failure rather than reset.
Main competitors: trend continuation when no real pullback exists, failed break when reclaim/failure logic dominates, breakout when compression release is the truer engine.

### C-03 — Breakout / Compression Release
Canonical idea: stored structure resolves through release from compression, balance edge, or boundary pressure.
Native habitat: compression, containment, pressure build, or poised expansion states.
Anti-habitat: already-mature trend continuation without real compression, or failed release that immediately reverses.
Main competitors: trend continuation, failed break, and balance rotation depending on whether release sustains, fails, or never truly escapes value.

### C-04 — Failed Break / Trap Reversal
Canonical idea: directional attempt fails, reclaims, and reverses as trapped participation is unwound.
Native habitat: rejected breakout, failed directional continuation, reclaim after rejection, or obvious trap logic.
Anti-habitat: clean successful breakout, stable trend continuation, or non-directional balance without a failed attempt.
Main competitors: breakout if release is still live, trend continuation if the original move is actually healthy, balance rotation if the case is not truly trap-driven.

### C-05 — Balance Rotation / Range Mean-Reversion
Canonical idea: value-dominant rotation within balance where excursion rejects and price returns toward acceptance/value.
Native habitat: balance, auction rotation, failed acceptance away from value, or repeated range rejection/return behavior.
Anti-habitat: genuine trend regime, genuine breakout release, or failure-driven reversal that is not value-centric.
Main competitors: breakout when balance is actually preparing to resolve, failed break when rejection becomes trap reversal, trend families when directional structure genuinely dominates.

## Family registry posture

The registry keeps four classes:
- core Aurora families
- conditional / extended families
- structure-preservable but not chart-only-native families
- future derivative/hybrid layers

The compiled wrapper hot path includes only the current core family set above.
Conditional and future classes remain source-truth references, not default wrapper doctrine.

## Family cards and doctrine fields the wrapper should preserve

For each live family, preserve:
- canonical thesis
- market habitat
- state and surface dependencies
- required data surfaces
- structural entry logic at high level
- structural invalidation logic at high level
- main anti-confusions
- common failure modes
- competitor relationships

## Current interpretation anchor

The canonical worked family example shows a C-04 vs C-03 vs C-01 contest where `C-04` survives as the best fit without pretending the case was perfectly clean.
That anchor matters because Aurora must preserve contested-primary behavior instead of backfilling certainty.
