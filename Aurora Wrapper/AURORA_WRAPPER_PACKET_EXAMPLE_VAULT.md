# AURORA WRAPPER PACKET AND EXAMPLE VAULT

## Purpose

This vault compiles the minimum concrete shapes and examples a wrapper needs when abstract protocol text is not enough.

## Packet family

### Workflow packet
Used to define the full staged Aurora flow for a case.
Must carry input requirements, stage order, object chain, output expectations, and operator notes.

### Family-lane packet
Used when a specific family/pattern lane needs a reusable operational route.
The active example lane is C-04 / G4 and includes habitat, competitors, required inputs, and stage-specific checks.

### Worked-example packet
Used to show a fully filled interpretation chain from ASC context through downstream judgment.
The active worked example demonstrates:
- honest ASC intake
- group-context construction
- context interpretation
- family and pattern narrowing
- opportunity posture
- geometry/card-ready logic

### Review packet
Used to compare original thesis against actual outcome without rewriting doctrine casually.
Must preserve original state summary, actual outcome, review classification, learned changes, and stability notes.

## Core object/schema anchors

### Group context object
Preserves identity, membership, regime, routing, and boundary fields for a grouped context.
It exists so family/pattern routing is not performed on a vague symbol narrative alone.

### Strategy-card field schema
Preserves identity, lineage, interpretation, family/pattern, deployability, geometry, timebox, invalidation, machine-safe export, and human-only commentary groups.

### Review packet schema
Defines how Aurora learns from a reviewed case without mutating earlier packet meaning retroactively.

## Wrapper use rule

Use this vault when you need shape certainty or interpretation anchoring.
Do not make it the first file read for simple control, family, pattern, or bridge questions.
