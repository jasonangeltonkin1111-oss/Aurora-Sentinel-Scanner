# AURORA WRAPPER PATTERN ROUTING MANUAL V1

## PURPOSE

This file deepens the wrapper's pattern knowledge so a GPT using only `Aurora Wrapper/` can reason more faithfully without relying on hidden Blueprint access.

It is not a replacement for:
- `AURORA_WRAPPER_PATTERN_VAULT.md`
- `AURORA_WRAPPER_EXECUTION_PACK.md`
- `AURORA_WRAPPER_FAMILY_VAULT.md`

It is a densification layer for wrapper-only use.

Its job is to preserve the pattern-routing intelligence that often gets flattened when source doctrine is compiled too aggressively:
- what pattern questions to ask first
- how patterns inherit meaning from family posture
- how stage changes pattern legitimacy
- how to preserve more than one live pattern candidate honestly
- how to avoid forcing pattern certainty too early
- how pattern validity differs from family validity

---

## ROOT LAW

Pattern routing is not label-matching.
It is the act of preserving the most truthful local structural expression **inside** a surviving family posture.

A wrapper-only GPT should therefore treat pattern routing as:
- downstream of family routing
- local but not family-blind
- stage-aware
- uncertainty-preserving
- anti-confusion-aware

not as a direct chart-shape lookup.

---

## PATTERN ROUTING QUESTIONS

When reasoning from wrapper files alone, the model should ask these pattern questions in order.

### 1. What surviving family posture owns the local structure?
Ask:
- which family or family set is still alive?
- is the family answer clear, contested, or deferred?
- is pattern routing even honest yet?

If family posture is still structurally open, pattern routing may need to stay deferred.

### 2. What local structural role is the case actually expressing?
Possible roles include:
- accepted break and hold
- orderly continuation pullback
- compression release
- failed break / reclaim
- range rejection / return to value
- none of the above honestly enough yet

### 3. What is the opportunity stage **inside** the pattern?
Ask:
- is the pattern early, developing, confirmed, continuation, re-entry, salvage, late, or exhausted?
- is this stage native for the pattern's parent family?
- is the pattern still valid even if the move is no longer early?

### 4. What anti-confusions matter most?
Ask:
- what else could this still be locally?
- is this a real accepted hold or just a spike?
- is this a healthy pullback or deterioration?
- is this real compression release or only noisy excursion?
- is this real reclaim hierarchy or unstable chop?
- is this real range rejection or emerging breakout?

### 5. What keeps the pattern alive, deferred, or invalid?
Ask:
- what still needs confirmation?
- what missing evidence prevents a clean promotion?
- what invalidation would kill the pattern without rewriting the family?

## PATTERN ECOLOGY G1 THROUGH G5

## G1 — Accepted Break and Hold

### Best use case
Use when the local structure expresses successful acceptance outside a prior boundary and the question is whether the hold is structurally real.

### Parent-family posture
Most native under:
- C-01 continuation
- C-03 breakout / compression release

### Native local meaning
- the break is not only visible
- the break is also holding
- the structure is expressing persistence outside prior containment
- the pattern depends on acceptance rather than mere excursion

### Strong stage legitimacy
Most legitimate in:
- `STAGE_CONFIRMED`
- early `STAGE_CONTINUATION`

Still potentially legitimate in:
- some `STAGE_MATURE` continuation contexts when hold quality remains strong

Usually weak in:
- `STAGE_LATE`
- `STAGE_EXHAUSTED`

### Main anti-confusions
- one-bar spike mistaken for acceptance
- late exhausted extension mistaken for healthy break-and-hold
- weak or reclaimed excursion mistaken for valid holding behavior
- failed break hierarchy not yet recognized

### Wrapper-only routing clue
If the move's local truth depends on whether price is **holding** beyond prior structure rather than merely crossing it, G1 should stay active.

---

## G2 — Orderly Continuation Pullback

### Best use case
Use when local structure expresses constructive retracement inside still-valid directional authority.

### Parent-family posture
Most native under:
- C-02 trend pullback / pullback continuation
- sometimes C-01 continuation

### Native local meaning
- directional authority already exists
- local retracement remains subordinate
- the pattern lives in restoration/resumption rather than fresh release
- pullback quality matters more than simple direction labels

### Strong stage legitimacy
Most legitimate in:
- `STAGE_DEVELOPING`
- `STAGE_CONTINUATION`
- `STAGE_REENTRY`

Still potentially legitimate in:
- `STAGE_CONFIRMED`
- some `STAGE_MATURE` continuation contexts

Usually weak in:
- `STAGE_LATE` when pullback has become deterioration
- `STAGE_EXHAUSTED`

### Main anti-confusions
- deterioration mistaken for healthy pullback
- reversal logic mistaken for subordinate retracement
- balance reform mistaken for continuation restoration
- unstable overlap mistaken for orderly pullback structure

### Wrapper-only routing clue
If the local question is whether retracement is still **constructive**, not whether trend once existed, G2 deserves serious attention.

---

## G3 — Compression Release

### Best use case
Use when local structure expresses release from meaningful compression or containment and the question is whether expansion is real.

### Parent-family posture
Most native under:
- C-03 breakout / compression release

### Native local meaning
- compression or containment mattered before the move
- local structure expresses release from that containment
- the pattern depends on actual pressure-resolution rather than random volatility

### Strong stage legitimacy
Most legitimate in:
- `STAGE_EMERGENT`
- `STAGE_DEVELOPING`
- `STAGE_CONFIRMED`

Still potentially legitimate in:
- early `STAGE_CONTINUATION`

Usually weak in:
- `STAGE_LATE`
- `STAGE_EXHAUSTED`

### Main anti-confusions
- noisy excursion mistaken for release
- price expansion with no real compression parent structure
- weak non-persistent release mistaken for real breakout behavior
- failed release not yet separated from valid release

### Wrapper-only routing clue
If the pattern's local truth depends on whether stored pressure has actually resolved, G3 should remain near the top.

---

## G4 — Failed Break / Reclaim

### Best use case
Use when local structure expresses failed release and reclaim hierarchy rather than continuing breakout authority.

### Parent-family posture
Most native under:
- C-04 failed break / trap reversal

### Native local meaning
- prior break or directional attempt mattered
- that attempt is now losing authority
- reclaim hierarchy is what gives the pattern meaning
- the local pattern depends on failure being structurally visible

### Strong stage legitimacy
Most legitimate in:
- `STAGE_DEVELOPING`
- `STAGE_CONFIRMED`

Still potentially legitimate in:
- `STAGE_SALVAGE`
- `STAGE_REENTRY`

Usually weak in:
- `STAGE_EMERGENT`
- `STAGE_EXHAUSTED`

### Main anti-confusions
- unresolved breakout drift mistaken for full failure/reclaim
- ordinary pullback mistaken for reclaim hierarchy
- unstable chop mistaken for clean failed-break structure
- rotational rejection mistaken for trap unwind

### Wrapper-only routing clue
If the local case only makes sense **after** acknowledging that the prior move has failed, G4 deserves real weight.

---

## G5 — Range Rejection / Return to Value

### Best use case
Use when local structure expresses rejection from a meaningful range edge or failed excursion back toward accepted value.

### Parent-family posture
Most native under:
- C-05 balance rotation / range mean-reversion

### Native local meaning
- the range/balance itself is real
- local excursion rejects
- the pattern's logic points back toward accepted value rather than further migration
- rotational structure dominates over breakout discovery

### Strong stage legitimacy
Most legitimate in:
- `STAGE_CONFIRMED`
- `STAGE_MATURE`

Still potentially legitimate in:
- `STAGE_LATE`
- `STAGE_REENTRY`

Usually weak in:
- `STAGE_EMERGENT`
- `STAGE_EXHAUSTED`

### Main anti-confusions
- emerging breakout mistaken for healthy rejection
- weak edge-touch mistaken for meaningful rejection
- trap reversal mistaken for orderly rotation
- range instability ignored while still calling the case rotational

### Wrapper-only routing clue
If the local structure makes most sense as rejection **back into value** rather than discovery away from value, G5 should stay active.

## PATTERN MULTI-CANDIDATE LAW

A wrapper-only GPT must not erase plausible competing patterns too early.

Common honest multi-pattern cases include:
- G1 vs G3 when both accepted continuation and fresh release expression remain plausible
- G3 vs G4 when release quality is still unresolved and failure/reclaim hierarchy may take over
- G4 vs G5 when rejection is visible but trap-unwind versus orderly rotation remains open
- G2 vs G4 when constructive retracement versus structural failure has not yet separated clearly

When this happens:
- preserve contested posture
- preserve what would resolve the contest
- do not force a single winner just to simplify later geometry

---

## PATTERN VALIDITY VS FAMILY VALIDITY LAW

Pattern invalidity does not always kill the family.

A case may preserve family truth while losing one local pattern expression.

Examples:
- C-01 continuation may survive even if G1 accepted hold weakens and the case becomes better expressed as G2 pullback continuation
- C-03 release may survive even if one early release expression fails before later structure resolves differently
- C-05 rotational family may survive even if one attempted edge rejection was too weak to validate G5 cleanly

Wrapper-only usage must preserve this distinction.
Do not rewrite the whole family because one local pattern failed.

---

## PATTERN DEFER AND REFUSAL LAW

Return `DEFERRED_PATTERN_COMPETITION` when:
- family posture is honest but still too unresolved for clean local pattern promotion
- pattern identity depends on missing local evidence
- breakout-versus-failure or continuation-versus-deterioration hierarchy is not yet clear enough
- the local structure is interesting but not yet pattern-stable

Return `NO_VALID_PATTERN` when:
- the family may still be alive but no local pattern expression is yet honest enough to preserve

Return `INVALID_INPUT` when:
- local structure/context identity is too broken even for bounded pattern routing
